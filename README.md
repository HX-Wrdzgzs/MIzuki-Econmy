# Amia Plugin Economy

Amia Plugin Economy 是 Amia / Mizuki 相关机器人生态中的经济与社交互动插件，当前处于新体系迁移与开发组内部维护阶段。

本仓库是个人名下的最新开发源仓库。组织侧仓库当前仍可能暂名为 `Amia-Mizuki-Dev-Team/Mizuki-plugin-economy`，后续如果组织仓库重命名为 `Amia-plugin-economy`，应以组织仓库作为主维护位置。

## 当前状态

- 维护状态：开发组内部维护中。
- 使用范围：面向开发组现有机器人环境与关联服务，不保证外部环境可直接运行。
- 文档状态：本 README 只描述项目定位、模块边界与维护注意事项，不提供公开部署教程或完整调用说明。
- 许可证状态：暂未选择开源许可证。未经明确许可，不建议再分发、商用或用于生产部署。

## 功能边界

该插件围绕群聊经济、虚拟资产、互动娱乐和图像化反馈构建，核心能力包括：

- 每日签到、打工、余额与个人信息展示。
- 背包、商城、道具、称号、兑换码等经济系统能力。
- 红包、转账、PK、猜拳、V50 报名等群内互动能力。
- 卡牌抽取、卡牌馆藏、合成、农场、宠物、副本、探索、成就等扩展玩法。
- 经济日志、每日任务、股市、拍卖行、日记墙等长期运行子系统。
- 面向现有机器人环境的图片渲染与 Markdown / 按钮消息适配。

这些功能依赖现有机器人运行环境、消息适配层、数据目录和外部服务状态。仓库代码不应被理解为独立可部署产品。

## 代码结构

主要文件如下：

- `__init__.py`：插件入口、命令注册、事件处理、启动初始化与部分业务编排。
- `config.py`：NoneBot 配置模型。
- `model.py`：经济系统数据读写、MySQL 交互、本地 JSON fallback 与核心业务操作。
- `draw.py`：签到卡片、个人信息、商城、背包、排行榜、卡牌等图像渲染逻辑。
- `const.py`：静态配置、道具表、卡牌/角色映射、任务、股市、农场、宠物、副本等常量。
- `crawl_cards.py`：PJSK 卡牌资源元数据与缩略图同步辅助脚本。

## 维护注意事项

- 目前入口文件仍然较重，命令注册、业务编排和部分状态管理集中在 `__init__.py`。后续维护时建议逐步拆分为 commands、services、repositories、renderers 等模块。
- 与数据库、图片缓存、卡牌资源、本地 JSON fallback 相关的行为应优先保持兼容，不建议在未验证现有数据的情况下大幅改动字段结构。
- 涉及余额、红包、转账、抽卡、兑换码、股市和拍卖等资金流或库存流的逻辑，应优先关注并发一致性和重复提交问题。
- 涉及外部资源地址、反代节点、机器人消息格式和按钮结构的改动，应先在开发组测试环境验证。
- 不要把真实数据库凭据、Token、Cookie、私有服务地址或运行时数据提交到仓库。
